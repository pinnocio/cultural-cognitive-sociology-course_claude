Preface

How This Course Is Organized

This course runs on three interlocking tracks across fifteen weeks:

Track A — Theory. The conceptual architecture of cultural and cognitive sociology: what the field claims, what it disputes, and where its active frontiers are.

Track B — Methods Laboratory. Operational implementation of the methods used to study cultural and cognitive phenomena. Every theoretical module has a corresponding lab. Code is written to run, not to illustrate.

Track C — Research Development. Four cumulative assignments scaffold a thesis-quality research design across the semester. The final deliverable (Assignment 4) is a complete research design paper, 7,000–10,000 words.

The course does not treat theory and methods as separable. Method selection is a theoretical act. Operationalization decisions carry ontological commitments. These connections are made explicit throughout.

Pedagogical Principles

  1. Constructs before measures. Every quantitative exercise begins with the theoretical construct, then asks how (and whether) it can be operationalized with the available data.

  2. Replication before innovation. Lab exercises begin by replicating published results. Only once a method is understood as implemented do students modify and extend it.

  3. Failure is data. When a model fits poorly, when a factor structure does not replicate, when a class solution is theoretically uninterpretable — this is informative, not a mistake to be hidden.

  4. Pre-registration as discipline. Assignment 3 requires a pre-registration document before analysis. This trains the habit of separating confirmatory from exploratory inference.

Prerequisites

Students should arrive with:

  • Introductory graduate statistics (regression, basic probability)
  • Working R installation (≥ 4.2.0)
  • Familiarity with tidyverse data manipulation

No prior SEM, LCA, or MCA is assumed. These are built from scratch in the methods labs.

Run the following to install all packages used in this course:

source("packages.R")

A Note on Reproducibility

All lab code sets a random seed explicitly. Results should be fully reproducible across machines given the same package versions. Session information is printed at the end of each chapter. If your results diverge, check package versions first.

R.version.string
